变量覆盖

可以用传参值替换程序原有的变量值

寻找变量覆盖：

  1. $$使用不当
  2. extract()函数使用不当：将数组中的键值对转换为变量
  3. parse_str()函数使用不当：将查询字符串解析到变量中
  4. import_request_variables()函数使用不当
  5. 开启了全局变量注册等等

session覆盖：session处理不当，会引发变量覆盖

  1. cookie 客户端生成，可能被伪造（xss打cookie）
  2. session 服务器端生成，理论下，session不可能被伪造
  3. token sign、JWT签名（json web token）time时间戳

代码审计

根据变量覆盖漏洞的特点，先搜索关键函数，没有得到相关有用信息
搜索可变变量（$$）
其中，被双引号引起来的，被当作字符串的不用看

common.php

遍历数组Array，依次取出其中的元素并赋值给变量$request
遍历变量request中的元素$_k(key：键)和$_v(value：值)，$_v用_RunMagicQuote方法处理，并赋值给$_v

_RunMagicQuote方法用于处理输入数据，确保数据在存入数据库前经过转义处理

如果变量名以”cfg_”或”GLOBALS”开头且未设置为cookie时，就会终止程序并输出’Request var not allow!’。

comment.php

查找引用了common.php的文件，发现comment.php文件在包含common文件前还开启了session会话，此时我们可以尝试去寻找session覆盖

全局搜索session，发现有session的函数调用，根据函数名猜测应该是保持用户的连接状态（其中包含的元素：用户id，组id，用户名），当用户id和组id不为空，则将信息存储在session中，并返回1，前面还有开启session会话的函数

用户鉴权以组id为主，全局搜索groupid，当组id值为1的时候为系统管理员权限，此时权限最高

$keepUserIDTag的值为”duomi_admin_id”;
$keepgroupidTag的值为”duomi_group_id”;
$keepUserNameTag的值为”duomi_admin_name”;

payload基本格式为：?_SESSION[duomi_admin_id]=1&_SESSION[duomi_group_id]=1&_SESSION[duomi_admin_name]=111

我们现在需要寻找可以将payload写进去的文件，要求为：

  1. 包含session_start();
  2. 能够包含common.php文件

comment.php文件能够符合要求127.0.0.1/interface/comment.php?_SESSION[duomi_admin_id]=1&_SESSION[duomi_group_id]=1&_SESSION[duomi_admin_name]=666
